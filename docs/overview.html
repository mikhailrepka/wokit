WO Kit

I'm happy to introduce to you my frontend toolkit, a lightweight framework called WO Kit, short for Webberin One Kit, part of my CMF/CMS platform — Webberin One.

Atomic UI

WO Kit contains css and js tools to create any UI you want. All css code is built using the atomic philosophy, where each element is an atom, each component contains many elements, each of which can be replaced or modified. Element starts with .__*, modifier starts with .--*.

The main css file contains variables for all settings: colors and sizes. Each of the following settings is obtained by calculations from the basic variables. That's why you can scale the interface by changing a small number of variables.

Example

Markup

<button class="__wo-btn --primary">Button example</button>

wo.css

:root {
--wo-white: #fff;
--wo-bg: #F4F4F4;
--wo-bg-dim: #EAE7E7;
--wo-black: #333333;

--wo-primary: #108BB3;
--wo-primary-dim: #0E7A9D;

--wo-font-size: 1rem;
--wo-border-radius-button: .25rem;
}

.__wo-btn {
	font-size: var(--wo-font-size);
	color: var(--wo-black);
	background: var(--wo-bg);
	padding: calc(var(--wo-font-size) * .5) calc(var(--wo-font-size) * 1);
	border: 1px solid var(--wo-bg-dim);

	border-radius: var(--wo-border-radius-button);
	-moz-border-radius: var(--wo-border-radius-button);
	-webkit-border-radius: var(--wo-border-radius-button);
}
.__wo-btn.--primary {
	color: var(--wo-white);
	background: var(--wo-primary);
	border: 1px solid var(--wo-primary-dim);
}

HTML5 doctype

Of course WO Kit requires the use of the HTML5 doctype.

<!doctype html>
<html>
	...
</html>

Viewpoint and mobile-first

WO Kit is a mobile-first driven framework, that's why you need to use meta element viewport like that:

<meta name="viewport" content="width=device-width, initial-scale=1, user-scalable=0">

Param user-scalable may be 0 (off) or 1 (on), for your taste. If it off — ui can't be scalable.

Bug report and community

WO Kit don't have a blog or froums, but you can report issues to github issues.
